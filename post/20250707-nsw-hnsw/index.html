基于图的向量搜索算法：从nsw到hnsw

2025年7月7日

概述 #

基于图的贪心算法向量搜索 #

如果把向量看成空间中的一个点，向量搜索就是在空间中寻找相近的点。

// 在空间中贪心算法搜索距离目标节点最近的节点返回
function search(target, space):
  // 随机选择空间中的一个点做为起始点start_node
  entry_point = select_random_point_from_space(space)
  // 集合保存已经访问过的节点集合
  visited = {}
  // 保存当前要遍历的节点
  candidates = {entry_point}

  while candidate:
    // 取出当前节点
    current = candidates.pop()
    // 得到当前节点的邻居节点集合
    neighbor_nodes = get_neighbor_nodes(current)
    // 邻居节点集合要排除已经被访问过的节点
    neighbor_nodes -= visited
    
    // 将当前节点保存到visited集合中，避免重复访问
    visited.append(current)
    
    // 当前节点和邻居节点组成一个节点集合
    nodes = {current} + neighbor_nodes
    
    // 求集合中距离目标节点距离最小的节点
    min_distance_node = min_distance_node_to_target(nodes, target)

    if min_distance_node == current:
      // 如果最小距离节点就是当前节点，返回
      return current
    else:
      // 否则，以最小距离节点继续搜索
      candidates += {min_distance_node}

我们考虑这样一个贪心算法去查找空间中距离一个点最近的点：

  • 随机从空间中选出一个点；
  • 计算该点和与它相邻的点与待搜索点的距离，如果该点与搜索点的距离比它的邻居都更小，就认为这个点是最近的点，返回；
  • 否则，选出相邻点集合中距离最近的点，重复以上查找步骤。

以下图为例讲解以上的流程，待查找的点为图中的红点，黑点为空间中当前已经存在的点：

  • 假设第一步随机选出来的起始点是I。
  • 对比I和I的邻居节点[B,C]，点B距离红点的距离最近，以B点开始下一轮的搜索。
  • B的邻居节点为[I,A,H,E]，这里要去掉已经遍历过的点I，对比B和去掉点I之后的邻居节点集合[A,H,E]，E点的距离更近，以E点开始下一轮的搜索。
  • E的邻居节点为[G,J,D]（去掉了已经遍历过的节点B），E距离待搜索点的距离比邻居节点更近，不再继续搜索，返回E点。

以上的算法看起来工作很良好，但是有几个问题：

  1. 找到的并不是最优结果，图中的点L比点E距离红点更近。
  2. 图中的点K是一个孤立节点，如果随机的起始节点是K，将得到一个更差的结果。
  3. 这里的问题只要求返回一个距离最近的点，如果要求返回大于2个，而图中的L和E并没有直接连接，将增加迭代的次数。

我们来逐个看上面提出的问题：

  1. 如果要解决问题1，保证每一次返回的都一定是最近的节点，要求整个图是一个连通图，插入新节点时的时间复杂度以及数据存储的空间复杂度就上来了。
  2. 如果不要求这么极端的考虑（即要求空间中所有点都互通），那么是否可以退而求其次：将空间划分为一个一个的子空间，即要求空间中：
  • 绝对不能出现孤立点，每个点都必须和其它点有连接，解决问题2。
  • 不要求所有点和空间中其它点都有连接，但是可以要求子空间中的点有连接，这样可以降低问题1出现的概率，同时也能解决问题2。

以上的思想就是NSW（Navigable Small World graphs）算法的核心思路：NSW将空间划分为小的子空间（所谓的“Small World graphs”），接下来看这个算法的具体实现。

NSW算法 #

构建NSW算法所需的图时，有参数控制子空间中点的数量（假设为m），我们来看新增节点时的流程：

  • 按照前面给出的贪心算法，在空间中寻找m个和新增节点最近的点做为新增节点的邻居节点。
  • 连接新增节点和邻居节点。

HNSW算法 #

理解了NSW算法之后，理解HNSW算法并不困难，HNSW算法可以认为是“NSW算法的跳表（skiplist）版本”。

我们首先简单回顾一下跳表的思想，如下图所示是跳表的结构：

  • 最下面的层次0存储了所有数据，数据之间是有序排列好的；
  • 0层以上的数据是指针，每一个指针指向一个比本节点更大的数字。

使用跳表搜索数据时，从最高层开始，对比指针和带搜索的数据：

  • 如果待搜索的数据更大，就到指针指向的位置继续搜索；
  • 否则，说明指针所在的数据范围已经包含了待搜索数据，那么将移动到下一层继续按照上面的流程搜索。

以图中为例，待搜索的数据是20，流程为：

  • 第3层指针指向的下一个数据是7，7 < 20，说明不在这个范围内，于是到达7所在的层次继续搜索；
  • 7所在的第3层指针是15，15 < 20，说明不在这个范围内，于是到达15所在的层次继续搜索；
  • 15所在的第2层指针是20，找到数据，返回查找结果。

可见，跳表的精髓是：以空间换时间，牺牲一些空间存储指针，通过指针的数据判断，跳过一些数据的对比，减少比较的次数。

HNSW本质上就是多层次的NSW，因此被称为HNSW（Hierarchcal Navigable Small World graphs），把它理解为把空间分为多个层次，每层上运行的还是NSW算法。

如下图所示是HNSW图的构造：

  • 每一层数据都是一个NSW图结构。
  • 层次越高，数据越稀疏；反之，层次越低，数据越密集。和跳表一样，第0层存储了所有数据。
  • 查找数据时，和跳表一样，也是从最高层开始找：
    • 从最高的第n层开始，使用NSW的贪心算法，找到距离待查找节点的最近节点。
    • 找到第n层的最近节点，顺着这个节点到下一层第n - 1层，继续上面的流程。
    • 以上的流程一直到第0层，找到距离节点最近的节点。
